1. Introduction

UltraTorrent is a clean, fast, multi-user web interface for your existing BitTorrent client. It does not download torrents itself — it controls an engine such as rTorrent and presents everything through one modern UI with real-time updates, role-based access control, and a full audit trail.

This manual covers the Community edition. Everything below works in the free, open-source build.

2. Signing in

Open the UltraTorrent URL in your browser and sign in with the username and password your administrator created.

  • Two-factor authentication (2FA): if enabled on your account, you'll be prompted for a 6-digit code from your authenticator app after your password.
  • Sessions: you stay signed in via a secure token that refreshes automatically. Use Sign out in the user menu (top right) to end your session on this device.

3. Getting around the interface

The app has three main areas:

  • Sidebar (left): grouped navigation — Overview, Torrents, Automation, Files & Media, Administration, and System. You only see the items your permissions allow. Collapse the sidebar to an icon rail with the toggle at the bottom; on mobile it opens as a drawer.
  • Top bar: breadcrumbs showing where you are, live download/upload rates, a realtime-connection indicator, and your user menu.
  • Main area: the current page.

The version number and an About UltraTorrent dialog are available from the sidebar footer and the user menu.

4. Dashboard

The Dashboard is your at-a-glance overview:

  • Current download and upload speeds and overall share ratio.
  • Total and active torrent counts, and a breakdown by state (downloading / seeding / completed / paused / errored).
  • A live bandwidth chart and a recent activity feed.
  • Engine status (online / offline).

5. Managing torrents

Adding a torrent

Click Add Torrent and choose one of three sources:

  • Magnet link — paste a magnet: URI.
  • Torrent file — upload a .torrent file.
  • Remote URL — an http(s) link to a .torrent.

Optionally set a Save path using the Browse picker (see §6). Paths are restricted to the folders your administrator allows.

The torrent list

The list shows name, progress, size, speeds, ratio, ETA, seeds/peers, and when it was added. You can:

  • Sort by any column and search by name.
  • Filter by state using the pills (All / Downloading / Seeding / Completed / Paused / Errors). The sidebar links to these filtered views too.
  • Select multiple torrents for bulk actions.

Actions

ActionWhat it does
Start / Stop / Pause / ResumeControl the transfer state.
RecheckRe-verify downloaded data against the torrent.
MoveRelocate the torrent's data to another allowed folder.
RemoveRemove the torrent, keeping its downloaded files.
Remove with dataRemove the torrent and delete its files. Irreversible — requires a separate permission and is always audited.

Details

Click a torrent to open its details drawer, with tabs for Overview, Files (per-file priority), Peers, and Trackers. Live data updates as it changes.

Categories & tags

Organize transfers with managed categories and tags.

6. The file manager

Under Files you can manage the files on your download volumes. Browsing is confined to the roots your administrator configures (the Default Root Path); you cannot navigate above them.

  • Browse folders, preview text files, and download files.
  • Create folder, rename, move, and copy.
  • Delete moves items to a Trash you can restore from or empty. Permanent deletes ask for confirmation.
  • The Cleanup Wizard finds junk/leftover files for review before removal.
  • Wherever a folder is needed, a Browse picker lets you pick a location instead of typing a path.

7. RSS automation

Under RSS Feeds you can subscribe to feeds and auto-download matches.

  • Add / edit a feed — name, URL (http/https), and refresh interval.
  • Rules — per feed, add rules with include / exclude patterns, an optional save path (via the picker), and an auto-download toggle. Edit a rule with the pencil button.
  • Match preferences — fine-tune how items are matched and prioritized, with a builder and test tools.
  • History — see what each feed has matched and downloaded.

8. Automation rules

Under Automation you can build "when this happens, do that" rules — for example, when a torrent completes, move it to a folder or send a notification. Each rule has a trigger, optional conditions, and one or more actions; rule activity is logged.

9. Notifications

UltraTorrent shows an in-app notification feed and can also fan events out to external channels (webhook, Discord, Slack, Telegram). Completed downloads notify automatically.

10. Search

Use Search for fast lookup across your torrents.

11. Your account & security

Open Account & security from the user menu to:

  • Change your password.
  • Enable two-factor authentication (2FA) — scan the QR code with an authenticator app, confirm a code, and save your one-time recovery codes somewhere safe.
  • Manage API keys — issue and revoke personal keys for scripting.

12. Administration

Administration features require the appropriate permissions; most users will not see them.
  • Users — create users and assign roles. Roles map to granular, dot-namespaced permissions (e.g. torrents.delete_data), from Super Admin down to Read-Only. Only a Super Admin can grant the Super Admin role.
  • Modules — view available modules and their status.
  • Settings — configure the instance. This includes the Default Root Path, which sets the highest folder users may browse to; browsing can only be narrowed within the server's allowed storage roots.

13. System & audit

  • Audit Log — a searchable record of security-relevant and destructive actions, showing who did what, when, from where, and the result.
  • Health and version information is available to administrators.
