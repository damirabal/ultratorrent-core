# Learn UltraTorrent

Everything here builds on what came before it. If you are new, read it in order — the
whole path is about an afternoon, and most of it is waiting for downloads.

## The path

| Step | Page | What you get |
| --- | --- | --- |
| 1 | [**Quick Start**](/learn/quick-start) | A running install and a finished download, in ~15 minutes. |
| 2 | [Core Concepts](/learn/concepts) | The vocabulary the rest of the docs assume: engines, indexers, watchlists, profiles. |
| 3 | [Architecture Overview](/learn/architecture-overview) | Which container does what, and which ports matter. |
| 4 | [My First Download](/learn/first-download) | The full lifecycle, one screen at a time — add, download, identify, organise, notify. |
| 5 | [Workflows](/learn/workflows) | How the pieces fit together end to end, as diagrams. |
| 6 | [Tutorials](/learn/tutorials/) | Beginner → enterprise, each one a complete working setup. |

:::tip Read Core Concepts even if you are in a hurry
Most support questions are really vocabulary questions. Ten minutes on
[Core Concepts](/learn/concepts) will save you an hour in
[Troubleshooting](/operate/troubleshooting).
:::

## Once you are comfortable

- **Configure a feature properly** → [Modules](/modules/) — one deep page per feature.
- **Run it seriously** → [Operate](/operate/) — security, backups, performance, maintenance.
- **Call it from your own code** → [REST API Reference](/reference/api).
- **Something is broken** → [Troubleshooting](/operate/troubleshooting), organised by symptom.
