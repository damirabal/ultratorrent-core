# Help

## Start with the symptom

If something is actually broken, go to [**Troubleshooting**](/operate/troubleshooting)
first. It is organised by *symptom* rather than by component, and every entry follows the
same shape: **Symptom → Diagnose (exact commands) → Root cause → Fix → Verify**.

## In this section

| Page | Use it when |
| --- | --- |
| [**FAQ**](/help/faq) | You have a question that starts with "can it…", "why does it…", or "should I…". |
| [**Glossary**](/help/glossary) | A term in the docs is doing work you do not recognise — *tconst*, *hardlink*, *Torznab*, *parking*. |

## Still stuck?

1. **Search** — press <kbd>Ctrl</kbd>/<kbd>⌘</kbd> + <kbd>K</kbd>. The whole site is indexed,
   and search runs offline.
2. **Check the reference** — [REST API](/reference/api), [Permissions](/reference/permissions),
   and [Environment Variables](/reference/environment) are generated from the source at build
   time, so they cannot be out of date.
3. **Open an issue** — [GitHub Issues](https://github.com/damirabal/ultratorrent-core/issues).
   Include the diagnostics the troubleshooting page told you to gather; it is the difference
   between a one-reply answer and a week of back-and-forth.
